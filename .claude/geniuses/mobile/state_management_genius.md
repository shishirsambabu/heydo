# State Management Genius — Predictable, testable app state

> Genius under the [Mobile Grandmaster](../../sovereigns/MOBILE_GRANDMASTER.md). Reports to that sovereign; ultimately serves the [Supreme Architect](../../SUPREME_ARCHITECT.md) and the [Trust Infrastructure Singularity](../../TRUST_INFRASTRUCTURE_SINGULARITY.md).

## Specialty
Predictable, testable app state.

## Mandate for Heydo
Establish predictable, testable state management so the app behaves correctly across offline/online transitions, gig lifecycle changes, and money states.

## Responsibilities
- Choose and standardize the state approach across the app.
- Model gig lifecycle, escrow, and verification states explicitly.
- Ensure state survives backgrounding, network loss, and restarts.

## Heuristics & principles
- Explicit state beats implicit guesswork, especially for money.
- If a state is possible, handle it — don't hope it won't happen.

## Deliverables
State management standard + lifecycle state models.

Phase 2, Phase 3.
