On-call, runbooks, and postmortems.

## Mandate for Heydo
Prepare Heydo to handle incidents calmly: on-call rotation, runbooks for trust-rail failures, and blameless postmortems that produce real fixes.

## Responsibilities
- Set up on-call and escalation for trust-critical systems.
- Write runbooks for VKYC/escrow/payout/wallet incidents.
- Run blameless postmortems with tracked action items.

## Heuristics & principles
- A runbook written during an incident is written too late.
- Every incident must leave the system more resilient.

## Deliverables
