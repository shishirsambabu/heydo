# Context — Competitors & Positioning

> Who else is in the space and why Heydo is different. Used to keep our differentiation sharp.

## Incumbents / adjacent players
- **Urban Company** — managed services, **dispatch** model, ~70% to workers, English-first, metro-focused, no applicant model, no Malayalam-first experience.
- **Generic dispatch / aggregator apps** — assign whoever is available; worker is interchangeable; no portable reputation; minimal worker protection.
- **Local classifieds / WhatsApp groups** — high trust friction, no verification, no escrow, no recourse.
- **Staffing agencies** — offline, opaque, slow, no digital identity or ratings.

## Where they are weak (our openings)
| Gap | Heydo's answer (USP #) |
|---|---|
| Dispatch removes giver control | Applicant model — apply + choose (#1) |
| No real identity verification | Live VKYC, Aadhaar-linked (#2) |
| Low worker payout (~70%) | 85% to workers (#4) |
| English-only | Malayalam-first (#5) |
| No payment protection | Full escrow (#6) |
| Workers unprotected | Dual-side ratings + safety + insurance (#7, #8) |
| No portable reputation | Heydo Score credential (#9) |
| Creative/lifestyle gigs unserved | Own that category first (#3) |
| No local moat | Kudumbashree + Malayalam network effect (#15) |

## Positioning
Heydo is **not** "Urban Company for Kerala." Heydo is the **trust infrastructure for the informal economy** — verified identity + portable reputation + safe payments — that happens to start as a gig marketplace. Compare us to a trust rail, not a dispatch app. See [category_creation_genius](../geniuses/investment/category_creation_genius.md).

## The copy threat & our defense
A well-funded incumbent could copy features, but **not** the 2-year VKYC trust graph, the Heydo Score that workers stake careers on, the Kudumbashree MoU, or the Malayalam network effect. Defense = deepen the moats every phase (see [moat_design_genius](../geniuses/investment/moat_design_genius.md)).
